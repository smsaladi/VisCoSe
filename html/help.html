Alignment
The alignment has to be in FASTA format like as follows (no other formats are supported):
   > defline1
   AMNWEQQMS...
   >defline2
   YYMSHHCWD...
   ...
Deflines have to start with a ">", and the following line(s) must consist of sequence data only. Deflines and sequence data have to be separated by CRs or CR/LFs.

 

Groups
Groups are used to simulate multiple datasets. Though VisCoSe is able to read multiple datasets using the commandline version, the VisCoSe website can handle one datasets at a time only. To enable the consensus sequence comparison feature for web usage of VisCoSe the user can define groups of sequences. The group definition is used to split the pasted or uploaded dataset into subsets. These subsets are then subjected to individual analysis by VisCoSe as if they were independant datasets. For example the group definition "1-26;27-35,43-50;36-42" defines the following groups:
 

   group 1: sequences 1 to 26
   group 2: sequences 27 to 35 and sequences 43-50
   group 3: sequences 36-42

For all defined groups, the sequence subset will be extracted. For each subset, VisCoSe generates a consensus sequence and a corresponding visualisation of it together with it's underlying alignment (if specified). The handling of the groups feature was implemented using the Bit::Vector-Library [1].

 

Alignment / Coloring options
These options deal with dataset/subset handling and coloring of the histogram visualising the conservation rates as well as the underling alignments:

- Force realignment of subsets after splitting even if the source dataset was already aligned
  If the dataset the user provided was already aligned and groups were defined, the default behaviour of VisCoSe is not to realign the resulting subsets but to preserve the original user alignment (default: inactive)
- Colorize the histogram in consensus colors
  When activated, the columns of the conservation rate histogram are shown color-coded according to their conservation rate (blue indicates 0%, cyan ~25%, green ~50%, yellow ~75% and red ~100% conservation) (default: active)
- Height of the conservation rate histogram
  Specifies the height of the histogram illustrating the conservation rate for each position of a consensus sequence (default: 33)
- Show no alignment (above consensus sequences)
  Activating this option prevents any visualisation of underlying alignments (default: inactive)
- Show alignment, plain formatted
  This option causes the underlying alignment to be shown on top of the consensus sequence, thus allowing visualization of subsets (default: inactive)
- Show alignment, color formatted (overrides 'plain formatted')
  As above, except that the alignment is colorized as follows:
if an amino acid of a sequence at position x is the same as the amino acid of the consensus at position x, the amino acid of the sequence is colorized according to the conservation rate. Activating this option overrides the option "plain formatted" of VisCoSe (default: active)
- Show alignment, color formatted for all amino acids (overrides 'color formatted', results in larger files)
  As above, except that each amino acid in the alignment is colorized according to its conservation rate, regardless of the amino acid of the consensus sequence at that position. Activating this option overrides the options "plain formatted" and "color formatted" (default: inactive)

 

Viewing options
The following options directly influence the appearance of the results being presented using HTML documents:

- Background color
  Specifies the background color for the HTML output. The web interface offers 5 pre-defined colors: white, lightgray, midgray, darkgray and black. By using the commandline version of VisCoSe the user is able to use any background as well as foreground (= text) color by specifying using HTML tags (default: white)
- Font size
  Specifies the font size for the HTML output. A size <6 makes the text hard to read but can be useful for providing an overview. For changing the font size "on-the-fly" for already calculated results, please use the "text zoom" option in your browser ("View->Text Zoom" for Mozilla, "View->Text Size" for Internet Explorer) (default: 8)
- Character used for conservation rate histogram columns
  Here the user may provide the character used by VisCoSe for "drawing" the histogram. In our opinion, "|" is most visually appealing, but other people may prefer another character like "#" (default: |)
- Linebreak at character x
  By activating this option and adjusting of the linebreak position, VisCoSe inserts line breaks at the specified position. The consensus sequence and the underlying alignments are then made up at position x, making printing of long alignments/consensus sequences more easy (default: off)

 

Alphabet options
Using the alphabet options, the user is able to additionally retrieve a VisCoSe result (consensus sequence and underlying alignment) for a chosen reduced amino acid alphabet:
- Alphabet list
  This box offers several chosen reduced amino acid alphabets, ranging from the most simplified "HP model" to more sophisticated alphabets like those from Murphy et al. (2000) or Li et al. (2003) (see here for references and descriptions). If the default alphabet "None / Default 20 AA" is chosen, VisCoSe will only calculate a consensus sequence and conservation rates for the standard alphabet consisting of the 20 amino acids found in proteins. If another alphabet is chosen, VisCoSe will translate the dataset(s) into the new (reduced) alphabet. Afterwards, another calculation is performed on the resulting reduced dataset, so that the user will get two results for each dataset/subset: one using the original 20 AA alphabet and another one using the reduced alphabet (default: "None / Default 20 AA")

All implemented alphabets are described here. If you think there is an important alphabet missing, please drop me a note together with a description of the alphabet. I will add reduced alphabets on such a "per-request" basis.

15.04.2004 BUGFIX: Fixed a weird bug in the alphabet section which caused all amino acids to be translated into only a single character. It's so weird because I'm pretty sure this was working 100% a few days before!!!

06.04.2004 CHANGED BEHAVIOUR: all alignmens/datasets read by VisCoSe are now being converted to uppercase before doing any fancy stuff on them (columns mixed with lower- and uppercase characters disturbed the correct consensus calculation, eg. if an alignment was made using DiAlign).
23.03.2004 ADDED FUNCTIONALITY: the FASTA-I/O-routine have been extended with better checking for format violations:
- handling multiple identical deflines: a warning is issued (to console or the logfile, respectively) and the n-th occurence of the defline is renamed to "defline_copyn".
- handling of non-standard characters (LF, CR, TAB)  in datasets: they are simply removed ;-) TABs in submitted datasets caused some "failures" of VisCoSe recently and disturbed the results (which were looking quite weird ;-)
ADDED FUNCTIONALITY: the logfile of a user's VisCoSe run is available on the results page (so each user has the possibility to look for errors / warnings for himself in more detail. I hope this lowers the "bug"-reports somewhat, because most of them were based on faulty FASTA format).
BUGFIX: I finally got rid of the absolute links in the results page. This was on the agenda for quite long, but was pushed back for more important and vital fixes.
